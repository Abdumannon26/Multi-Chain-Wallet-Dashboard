# Multi-Chain Wallet Dashboard

Дашборд для просмотра баланса и истории транзакций по адресу кошелька в выбранной блокчейн-сети (EVM, Solana). Angular 17+, standalone-компоненты, Angular Material.

## Как запустить проект

### Требования

- Node.js 20+
- npm или yarn

### Установка и запуск

1. Клонируйте репозиторий и перейдите в каталог проекта:

```bash
cd multi-chain-wallet-dashboard
```

2. Установите зависимости:

```bash
npm install
```

3. Запустите бекенд API локально (порт **3000**). Без запущенного бекенда запросы к портфолио и транзакциям будут завершаться с ошибкой.

4. Запустите приложение:

```bash
npm start
```

или

```bash
ng serve
```

5. Откройте в браузере: [http://localhost:4200](http://localhost:4200).

### Сборка

```bash
npm run build
```

Артефакты появятся в каталоге `dist/`.

### Тесты

```bash
npm test
```

Unit-тест для `ChainService`: `src/app/core/services/chain.service.spec.ts`.

### Конфигурация API

URL бекенда задаётся в `src/environments/environment.ts` (поле `apiUrl`). По умолчанию: `http://localhost:3000/api`. Для другого порта или окружения измените этот файл или добавьте `environment.prod.ts`.

## Какие решения были приняты и почему

### Стек и архитектура

- **Angular 17+, standalone-компоненты** — без NgModules, все компоненты и пайпы объявлены как `standalone: true`. Упрощает дерево зависимостей и lazy loading при необходимости.
- **Отдельный сервис на область** — `ChainService`, `PortfolioService`, `TransactionService` в `core/services/`. Один источник правды для каждого API, проще тестировать и переиспользовать.
- **TypeScript strict mode** — в `tsconfig.json` включены `strict: true`, `strictTemplates`, `strictInjectionParameters`. Меньше ошибок в рантайме, явная типизация контрактов и шаблонов.

### RxJS и отписки

- **Потоки вместо ручных подписок в UI** — в дашборде данные приходят через `Observable` (`portfolioState$`, `transactionsState$`, `showResults$`), в шаблоне используется `async` pipe. Отписка происходит при уничтожении компонента без ручного `unsubscribe`.
- **combineLatest + switchMap** — выбор сети и адреса объединены в один триггер; при смене пары (chain, address) предыдущий запрос отменяется, уходит один новый (portfolio и transactions запрашиваются параллельно).
- **takeUntilDestroyed(DestroyRef)** — в компоненте формы подписка на `getChains()` отменяется при уничтожении компонента. В дашборде все подписки на внутренние потоки тоже завершаются через `takeUntilDestroyed`.
- **shareReplay(1)** в `ChainService.getChains()` — список сетей запрашивается один раз и переиспользуется всеми подписчиками (форма и др.).

### Структура и компоненты

- **Форма выбора сети и адреса вынесена в отдельный компонент** (`WalletAddressFormComponent`) — своя зона ответственности, реактивная форма, валидация EVM/Solana внутри. Родитель только получает событие с выбранной парой (chainId, address).
- **История транзакций в отдельном компоненте** (`TransactionListComponent`) — приём `Observable<TransactionsState>` через `input()`, отображение загрузки/ошибки/таблицы и пустого списка. Дашборд не знает деталей вёрстки таблицы.
- **Пайпы для форматирования** — `formatBalance` (raw + decimals → строка с max 6 знаков) и `formatUsd` (число → валюта с 2 знаками). Использование в шаблоне через пайпы упрощает компоненты и даёт единообразное отображение.

### Валидация и ошибки

- **Валидация адресов в утилитах** (`address.util.ts`) — EVM: ровно 42 символа, `0x` + 40 hex; Solana: 32–44 символа, base58 (без 0, O, I, l). Проверка по типу сети из выбранной цепи. Сообщения об ошибках показываются в форме через `mat-error`.
- **Обработка ошибок API** — в дашборде все запросы обёрнуты в `catchError`. Сообщение пользователю формируется из `HttpErrorResponse` (сеть недоступна, 5xx, 4xx, тело ответа). Приложение не падает, показывается текст ошибки в карточке портфолио или транзакций.
- **Пустые и пограничные случаи** — пустой массив транзакций и отсутствие токенов обрабатываются в UI (сообщение «Транзакций не найдено», блок токенов не рендерится при пустом списке).

### UI и вёрстка

- **Angular Material** — карточки, форма (mat-form-field, mat-select, mat-input), кнопки, таблицы, спиннер. Единый вид и доступность без собственной реализации компонентов.
- **Адаптивная вёрстка** — форма переходит в один столбец на узких экранах; таблицы в горизонтальном скролле (`.table-scroll`), сохранены читаемость и удобство на мобильных.
- **Семантическая разметка** — `main`, `header`, `section`, `article`, заголовки и `aria-labelledby` для секций и форм.

### Форматирование чисел

- **Балансы** — всегда из `rawBalance` и `decimals` (BigInt), без приведения к float, чтобы не терять точность. В пайпе: целая и дробная части, не более 6 знаков после запятой, разделитель тысяч через `Intl.NumberFormat`.
- **USD** — два знака после запятой через `Intl.NumberFormat` (style: currency, USD).

### Тесты

- **Один unit-тест для сервиса** — `ChainService`: проверка GET-запроса к `/api/chains`, формата ответа и поведения `shareReplay` (один запрос при нескольких подписчиках). E2E по условию не делались.
